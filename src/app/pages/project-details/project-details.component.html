Project Management Tool

Ce projet a été réalisé dans le cadre de ma formation, dans le but de mettre en pratique les compétences techniques et les méthodes de gestion de projet acquises. L'objectif principal était de développer une application web permettant de gérer des projets, des tâches et des utilisateurs de manière intuitive et efficace. Cette application cible les équipes de travail qui souhaitent organiser leurs activités et suivre l'avancement de leurs projets.


Problème résolu

L'application vise à résoudre le problème de manque de visibilité et d'organisation dans la gestion quotidienne des projets. Elle propose une solution centralisée pour créer des projets, assigner des tâches aux membres d’équipe, suivre leur avancement et gérer les priorités.


Mon rôle

Dans ce projet, j'ai occupé plusieurs rôles :

  • Conception : Identification des entités principales (utilisateurs, projets, tâches) et création d'une architecture modulaire.
  • Développement : Implémentation de l'interface utilisateur en Angular, développement des API REST avec Spring Boot et mise en place d'une base de données relationnelle.
  • Tests et Industrialisation : Réalisation de tests automatisés (frontend et backend), dockerisation des composants, et mise en place d'une chaîne CI/CD pour garantir une livraison continue.

Objectifs atteints
  • Conception et modélisation d'une base de données relationnelle robuste.
  • Développement complet d'une application web avec une interface intuitive et des fonctionnalités backend performantes.
  • Mise en place d'un processus de tests automatisés avec une couverture de code à hauteur de 60% pour les instructions et les branches.
  • Intégration et déploiement continu (CI/CD) avec Docker et GitHub Acti

Détails Techniques

Choix technologiques :

  • Frontend : Angular a été choisi pour sa capacité à construire des interfaces utilisateur dynamiques et modérnes. Son architecture basée sur les composants facilite la maintenance et l'évolutivité.
  • Backend : Spring Boot a été utilisé pour sa robustesse et son écosystème complet, permettant une gestion facile des API REST.
  • Base de données : Une base relationnelle (PostgreSQL) a été choisie pour sa fiabilité et sa capacité à gérer des données structurées.
  • Tests : Utilisation de Jasmine/Karma pour le frontend et JUnit pour le backend.
  • Industrialisation : Docker pour la conteneurisation, GitHub Actions pour l'automatisation des tests et du déploiement.

Défis rencontrés
  • Gestion des relations complexes dans la base de données (ex. : relations utilisateur-tâches, tâches-projets) :
  • Solution : Conception d'un schéma de base de données bien normalisé et utilisation de JPA/Hibernate pour la gestion des relations.
  • Synchronisation des données entre le frontend et le backend :
  • Solution : Mise en place de services Angular avec gestion des observables (à l'aide de RxJS) pour un transfert de données fluide.
  • Automatisation des tests et couverture de code :
  • Solution : Configuration de pipelines CI/CD avec GitHub Actions pour l'exécution des tests automatisés et génération de rapports de couverture via Jacoco et Istanbul.

Solutions apportées
  • Une documentation claire du processus de déploiement dans un fichier README.md.
  • Dockerisation complète des services pour garantir un déploiement rapide et cohérent.
  • Mise à disposition des images Docker sur Docker Hub pour un accès public.

Résultats finaux
  • Une application fonctionnelle avec des fonctionnalités clés telles que la gestion des utilisateurs, la création de projets et la supervision des tâches.
  • Un processus industrialisé permettant un déploiement rapide et fiable, tout en garantissant la stabilité du code grâce aux tests automatisés.

Game Up

Ce projet a été réalisé dans le cadre de mes fonctions en tant que développeur pour la plateforme GamesUP, une plateforme de vente de jeux de société en ligne. L'objectif était de remédier à des problèmes critiques de performances et de fiabilité de l'API backend, qui impactaient directement les ventes. Le projet comportait plusieurs volets : refonte de l'API backend, mise en place de la sécurité, tests, et intégration d'un système de recommandation basé sur le Machine Learning.


Problème résolu

Le site était confronté à des bugs récurrents liés à un backend obsolète et mal conçu par un stagiaire. Ces problèmes rendaient la plateforme instable et faisaient perdre des ventes. Par ailleurs, le système de recommandations était aléatoire et inefficace, ce qui réduisait l'engagement des utilisateurs. Ce projet visait à redéfinir complètement l'architecture de l'API backend tout en intégrant un système de recommandation basé sur le Machine Learning.


Mon rôle

En tant que développeur principal, j'ai été en charge des tâches suivantes :

  • Refonte de l'API backend : Analyse et restructuration du code existant, mise en place d'une architecture REST modulable, et respect des principes SOLID.
  • Sécurité : Implémentation de Spring Security pour garantir l'intégrité des données.
  • Tests : Conception et exécution de tests unitaires et d'intégration pour assurer la stabilité et la fiabilité de l'API.
  • Système de recommandation : Finalisation d'une API Python avec FastAPI pour gérer les recommandations basées sur un algorithme KNN.
  • Documentation : Rédaction de documents techniques incluant les diagrammes d'architecture, de classes, de composants, et de séquences.

Objectifs atteints
  • Révision complète de l'API backend pour garantir sa fiabilité et sa performance.
  • Intégration d'un système de sécurité robuste avec Spring Security.
  • Mise en place d'un système de recommandation efficace connectant l'API Spring à une API Python.
  • Livraison de tests unitaires et d'intégration avec des rapports de couverture.
  • Documentation complète et précise pour assurer une compréhension globale du projet.

Détails Techniques

Choix technologiques :

  • Backend : Spring Boot a été choisi pour sa robustesse, sa flexibilité et sa compatibilité avec Hibernate pour la gestion des bases de données.
  • Sécurité : Spring Security a été utilisé pour la gestion des rôles (client et administrateur) et la protection des endpoints sensibles.
  • Recommandations : FastAPI a été préféré pour sa légèreté et sa facilité d’intégration avec des bibliothèques de Machine Learning telles que scikit-learn.
  • Tests : Utilisation de JUnit pour les tests backend et Pytest pour les tests Python.
  • Communication entre APIs : Mise en place de RESTful services pour la communication entre l’API Spring et l’API Python.

Défis rencontrés

    Code existant mal conçu :

  • Solution : Refactorisation complète pour respecter les principes SOLID et adoption d'une architecture modulaire.

    • Système de recommandation sans données fiables :

  • Solution : Génération de données synthétiques pour tester et valider le modèle KNN, en attendant des données réelles.

    • Intégration API Spring et FastAPI :

  • Solution : Mise en place de endpoints REST clairs pour échanger les données utilisateur entre les deux APIs.

Solutions apportées
  • Implémentation des opérations CRUD pour les entités principales (client, jeu, éditeur, commande).
  • Mise en place d'un système de rôles avec accès restreint pour les administrateurs.
  • Dockerisation des deux APIs (Spring et FastAPI) pour garantir un déploiement rapide et sans conflit.
  • Tests approfondis pour assurer la stabilité et identifier les régressions.

Résultats finaux
  • Une API Spring robuste, modulable et conforme aux standards REST.
  • Un système de recommandation fonctionnel basé sur l'algorithme KNN, prêt à être amélioré avec des données réelles.
  • Une documentation technique complète avec des diagrammes d'architecture, de classes, de composants et de séquences.
  • Une couverture de tests élevée et des rapports de qualité logicielle.

Electricity Billing System

Ce projet a été réalisé dans un cadre personnel afin de renforcer mes compétences en développement logiciel. L'objectif principal était de mettre en pratique mes connaissances en Java, MySQL et Swing en développant une application complète avec une interface utilisateur et une base de données.


Problème résolu

Le projet vise à résoudre un problème spécifique : la gestion efficace des données et leur affichage sous une interface conviviale. Il permet d'organiser et de manipuler des informations de manière structurée tout en offrant une interaction fluide avec l'utilisateur.


Mon rôle

Mon rôle dans ce projet était à la fois celui du développeur et du concepteur. Les objectifs atteints incluent la mise en place d'une application fonctionnelle, la connexion réussie entre l'interface utilisateur et la base de données, ainsi que la gestion des opérations CRUD (Create, Read, Update, Delete).

  • J'ai assuré la conception de la base de données
  • le développement du code en Java, ainsi que l'intégration de l'interface graphique avec Swing

Objectifs atteints
  • la mise en place d'une application fonctionnelle
  • la connexion réussie entre l'interface utilisateur et la base de données,
  • la gestion des opérations CRUD (Create, Read, Update, Delete).

Détails Techniques

L'application a été développée en utilisant les technologies suivantes :

  • Java pour la logique métier et la gestion des événements.
  • Swing pour la conception de l'interface graphique.
  • MySQL pour le stockage des données, avec JDBC pour assurer la connexion entre la base de données et l'application.

Défis rencontrés et Solutions apportées
  • Connexion à la base de données : Au début, j'ai rencontré des difficultés pour établir la connexion entre l'application et MySQL. Ce problème a été résolu en configurant correctement le driver JDBC et en optimisant les requêtes SQL.
  • Gestion des performances : Certains affichages étaient lents à cause du traitement des données. J'ai optimisé les requêtes et mis en place des index dans la base de données pour accélérer l'exécution des requêtes.
  • Interface utilisateur dynamique : L'utilisation de Swing demandait une gestion précise des événements et des mises à jour dynamiques de l'affichage. J'ai utilisé des modèles de conception comme MVC (Modèle-Vue-Contrôleur) pour améliorer la structure du code et faciliter la maintenance.
